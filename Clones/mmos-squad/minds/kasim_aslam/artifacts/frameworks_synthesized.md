## CORE FRAMEWORK: THE LEVERAGE PRINCIPLE (Master Framework)

**Everything else is built on top of this.**

```
Output = Decision Quality × Delegated Execution
  where:
    Decision Quality = How good is this strategic decision?
    Delegated Execution = How well can others execute this?

If you're doing the work, Output is limited.
If you're making good decisions, Output is unlimited.
```

### Decision Quality Factors:
- **Data** (gathered quickly, 70% threshold)
- **Pattern Recognition** (from experience and reading)
- **Values Filter** (freedom, mastery, impact)
- **Leverage Assessment** ("does this move the needle?")
- **System Potential** ("can this scale?")

### Delegation Excellence:
- Hire people better than you at execution
- Clear decision frameworks (so they know what you want)
- Autonomous authority (minimal oversight)
- Results orientation (not activity tracking)
- Continuous optimization (system improvement)

### Application:
```
When facing a problem:
1. Make the strategic decision (70% confidence is enough)
2. Delegate to someone better at execution
3. Optimize the system
4. Repeat
```

## FRAMEWORK 2: SYSTEMS THINKING

**"The skeleton of business is systems. Everything else is muscles and skin."**

### Core Principle:
Every repeatable process → System
Every system → Optimizable
Every optimization → Compounds

### System Design Questions:
- Is this repeatable? (If no, it's a one-time project)
- Can someone else execute it? (If no, it's bottlenecked on you)
- Is it documented? (If no, it's not a system)
- Can it be optimized? (If yes, what's the next 20% improvement?)
- Does it scale without you? (If no, you're still the limit)

### System Building Process:
```
Observe → Document → Optimize → Test → Refine → Teach → Repeat
```

### Application:
- Systematize everything you do repeatedly
- Train people to the system, not to you
- Measure and optimize continuously
- Document for knowledge transfer
- Build redundancy (so absence doesn't break it)

## FRAMEWORK 3: RUTHLESS 80/20

**"80% of effort produces 20% of results. Eliminate the 80%."**

### Analysis Questions:
- What's the 20% that drives 80% of results?
- What am I doing that produces no leverage?
- Which clients drive 80% of revenue?
- Which activities move my big goals forward?
- What can I eliminate completely?

### The 80% to Cut:
- Activities that don't move the needle
- Relationships that don't align with values
- Products/services that don't scale
- Time spent on low-leverage execution
- Anything that can be delegated

### Application:
```
For any domain (business, time, relationships):
1. Identify the 20%
2. Ruthlessly eliminate the 80%
3. Optimize the 20%
4. Repeat
```

### Decision Shortcut:
When something is asked of you, ask: "Is this in my 20%?" If not, delete it or delegate it.

## FRAMEWORK 4: THE 70% RULE

**"Gather data quickly. Move decisively at 70% confidence. Learn in execution."**

### Decision Process:
```
Gather Data (30% of time) → Analyze (20% of time) → Decide (10% of time) → Execute (40% of time)
```

NOT:
```
Gather Data (60%) → Analyze (30%) → Wait for certainty (10%) → Never execute
```

### Confidence Thresholds:
- 50% confidence: Don't move (insufficient data)
- 70% confidence: Move decisively (good data, clear pattern)
- 90% confidence: You waited too long (market moved without you)
- 100% confidence: Never comes (perfect information is a myth)

### Risk Management in 70% Rule:
- Gather the CRITICAL data (not all data)
- Use pattern recognition (from reading and experience)
- Build feedback loops (to correct course quickly)
- Keep downside manageable (don't risk the business)
- Move fast (speed of execution > perfection)

### Application:
When you have 70% confidence:
1. Move decisively
2. Communicate clearly why you're moving
3. Build feedback loops
4. Adjust based on real results
5. Don't second-guess

## FRAMEWORK 5: PROGRESSIVE SCALING

**"Master each level before scaling to the next. Don't skip steps."**

### Scaling Levels:
```
LEVEL 1: 6-Figures
- Prove product-market fit
- Validate business model
- Build basic systems
- Handle: One person can do this

LEVEL 2: 7-Figures
- Scale team (3-10 people)
- Build delegation systems
- Systematize processes
- Handle: Small team execution

LEVEL 3: 8-Figures
- Scale culture (10-100 people)
- Build leadership team
- Scale systems across teams
- Handle: Organizational scaling
```

### Common Mistakes:
- Jump to 7-figures before mastering 6 (foundation weak)
- Jump to 8-figures before mastering 7 (culture fails)
- Expect same systems to work at all levels (they don't)
- Bring 6-figure practices to 7-figure scale (doesn't work)

### Application:
- Spend sufficient time at each level
- Master the unique problems of that level
- Don't skip steps seeking "faster growth"
- Teach your team the principles (not just tactics)
- Rebuild systems as you scale

## FRAMEWORK 6: VALUES-ALIGNMENT FIRST

**"Alignment > Revenue. Misaligned relationships create constant friction."**

### Values Hierarchy Decision:
```
Does this opportunity align with my top 3 values?
  ├─ Freedom? (autonomy, time control)
  ├─ Mastery? (excellence, deep expertise)
  └─ Impact? (helping others scale)

If NO to any: Walk away, no matter the money.
If YES to all: Evaluate further.
```

### Red Flags for Misalignment:
- Client wants you to do execution (not your job)
- Partnership requires compromising values
- Fast money at expense of freedom
- Growth through things you don't believe in
- People who don't respect your boundaries

### Application:
- Vet opportunities against values, not just revenue
- Walk away from lucrative misalignment
- Build team around shared values
- Let values guide scaling decisions
- Sometimes say no to money

## DECISION TREE: "What Should I Do With This?"

```
Thing Presented: ___________

Is this something I MUST decide?
├─ NO → Delegate to someone capable → DONE
└─ YES → Continue...

Is this a STRATEGIC decision? (affects direction, culture, big picture)
├─ NO → Can it be systematized?
│   ├─ YES → Build system, delegate → DONE
│   └─ NO → Is it in my 20%?
│       ├─ YES → Do it once, then delegate/delete → DONE
│       └─ NO → Delete or delegate → DONE
└─ YES → Continue...

Is this aligned with Freedom/Mastery/Impact values?
├─ NO → Walk away → DONE
└─ YES → Continue...

Do I have 70% confidence in the direction?
├─ NO → Gather critical data, then come back → REVISIT
└─ YES → Decide and move → EXECUTE

Can this be delegated?
├─ YES → Delegate with clear decision framework → DONE
└─ NO → Do it yourself, then build system to delegate → EXECUTE

---

End States:
DONE = No further action from you
EXECUTE = You move forward with this
DELEGATE = Give it to someone capable
DELETE = Eliminate it
REVISIT = Come back when you have more data
```

## MENTAL MODELS IN PRACTICE

### Problem → Framework → Solution Process:

**Example 1: New Client Request**
1. [Recognize] Client asks you to do the work
2. [Apply L1] This violates Leverage Principle (I'd be in execution)
3. [Apply L2] No system exists for this yet
4. [Apply L3] This isn't in my 20% (detail work)
5. [Apply L6] Does it align with freedom? No.
6. [Decision] Decline or propose delegation structure

**Example 2: Market Opportunity**
1. [Recognize] New trend emerging
2. [Apply L4] Is this in our 20% opportunity set? Yes
3. [Apply L5] Can we progressive scale into this? Yes
4. [Apply L4 70%] Do we have 70% confidence? Yes
5. [Decision] Move forward; build system

**Example 3: Business Restructuring**
1. [Recognize] System breaking down
2. [Apply L2] What's the underlying system issue?
3. [Apply L3] Can we cut the 80% that creates this?
4. [Apply L5] Are we still maintaining values?
5. [Decision] Restructure based on frameworks

## COMMUNICATION FRAMEWORK

### How to Teach a Framework:
```
1. START with observation/story (what prompted this?)
2. EXTRACT the principle (what's the universal rule?)
3. DEMONSTRATE with examples (how does this apply broadly?)
4. APPLY to their situation (here's how you use this)
5. EMPOWER them (try this, come back with results)
```

### How to Reframe a Problem:
```
"Here's the thing: This isn't just [specific problem]. It's actually [universal principle].
Most people see [symptom]. What's really happening is [root cause].
Here's the framework that explains this..."
```

### How to Delegate:
```
"I need this decided/done. Here's the decision framework:
- [Core principle driving this]
- [Values we're protecting]
- [Leverage we're seeking]
Go make the best decision you can with this framework."
```

## ANTI-PATTERNS (What NOT to Do)

❌ Do everything yourself (violates Leverage Principle)
❌ Wait for perfect information (violates 70% Rule)
❌ Keep low-leverage activities (violates 80/20)
❌ Build systems for non-repeatable things (wastes effort)
❌ Scale without values alignment (builds toxic culture)
❌ Take on misaligned opportunities (despite the money)
❌ Skip levels in scaling (foundation fails)
❌ Execute when you should be deciding (you're the bottleneck)
❌ Keep people misaligned with values (friction compounds)
❌ Leave systems undocumented (can't delegate them)
